„Meine Website ist plötzlich komplett weiß.“

Keine Fehlermeldung.  
Keine Inhalte.  
Nur eine **leere Seite**.

Dieses Problem wird oft **White Screen of Death** genannt und tritt besonders häufig bei WordPress-Websites auf.

## Typische Ursachen

- Plugin-Konflikte
- Fehler im Theme
- PHP-Fehler im Code
- Serverprobleme
- Speicherlimit überschritten

Oft passiert das **nach Updates oder Änderungen** an der Website.

## 1) Plugin-Konflikt

**Problem:**  
Ein Plugin verursacht einen fatalen Fehler.

**Fix:**  
Plugins gezielt deaktivieren und Konflikt identifizieren.

Blindes Deaktivieren aller Plugins ist selten die beste Lösung.

## 2) Fehler im Theme

**Problem:**  
Ein Theme oder Child-Theme enthält fehlerhaften Code.

**Fix:**  
Theme-Dateien prüfen und Fehler isolieren.

## 3) PHP Memory Limit erreicht

**Problem:**  
Die Website benötigt mehr Server-Ressourcen als verfügbar sind.

**Fix:**  
Speicherlimit erhöhen oder problematische Funktionen entfernen.

## 4) Fehlerhafte Code-Anpassungen

**Problem:**  
Ein kleiner Codefehler kann die komplette Seite blockieren.

**Fix:**  
Logs analysieren und Fehler gezielt korrigieren.

## Warum die Seite einfach „weiß“ bleibt

Viele Server zeigen **keine Fehlermeldung im Frontend**, um Sicherheitsrisiken zu vermeiden.

Der eigentliche Fehler steht meist nur in:

- Server-Logs
- PHP-Logs
- Debug-Ausgaben

## Was du jetzt tun solltest

1. Keine weiteren Änderungen durchführen  
2. Fehlerquelle identifizieren  
3. Website stabilisieren

Je länger eine Website offline ist, desto größer können Ranking- und Kundenverluste werden.

## Wann sich ein Fix lohnt

Wenn deine Website:

- plötzlich nur noch weiß ist
- Inhalte nicht mehr lädt
- nach Änderungen nicht mehr funktioniert

## FAQ

### Ist meine Website komplett kaputt?

Meist nicht.  
In vielen Fällen ist nur **eine einzelne Komponente** betroffen.

### Wie schnell lässt sich der Fehler beheben?

Viele White-Screen-Probleme lassen sich **innerhalb weniger Stunden** lösen.
